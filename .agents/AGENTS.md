
# APASIFIC Development Rules

1. **Documentation First**: Always document architecture and design decisions before writing code.
2. **Architecture Freeze Before Coding**: Ensure the architecture is fully approved and frozen before starting implementation.
3. **One Sprint = One Deliverable**: Focus on a single milestone per sprint.
4. **No UI Changes Without Approval**: Do not alter UI components or pages without explicit user consent.
5. **No Database Changes Without Approval**: Any schema or database modifications require prior approval.
6. **No New Module Outside Approved Architecture**: Stick strictly to the defined and approved architecture.
7. **Every Sprint Ends with Documentation + Review**: Summarize work and request review at the end of each sprint.
8. **Implementation Starts Only After Sprint Approval**: Execution phase only begins when the sprint plan is explicitly greenlit.
